VPS Hardening Suite

Idempotent, modular Bash automation that transforms a bare Ubuntu/Debian VPS into a hardened, fully observable production server. Installs and wires together a multi-layer security stack (UFW, Fail2Ban, CrowdSec) and a complete observability stack (Prometheus, Grafana, Loki, Node Exporter, cAdvisor, Promtail) in a single command.

Ubuntu 20.04+ / Debian 11+ Idempotent Non-interactive mode All monitoring on localhost

рҹҸ—пёҸSystem Architecture

Traffic flows through four hardened layers before reaching any application. The monitoring stack runs entirely in Docker on an isolated bridge network, with all ports bound to 127.0.0.1.

рҹӣЎпёҸSecurity Layers

  • 1

    Network вҖ” UFW + CrowdSec

    Default-deny inbound policy at the kernel netfilter level. CrowdSec blocks community-flagged IPs before they touch any service. Docker traffic controlled via the DOCKER-USER iptables chain.

  • 2

    Authentication вҖ” SSH + Fail2Ban

    Password authentication disabled. ed25519 + RSA host keys, modern ciphers only, MaxAuthTries=3, agent/TCP forwarding disabled. Fail2Ban watches auth.log and bans brute-force IPs after 3 failures.

  • 3

    Runtime вҖ” Docker Isolation

    Containers run with least privilege. json-file log driver capped at 10 MB / 3 rotations. Live-restore enabled. No container port exposed on 0.0.0.0.

  • 4

    Observability вҖ” Loki + Promtail

    All security events (auth, UFW, Fail2Ban, CrowdSec) shipped to Loki within seconds. Full audit trail queryable in Grafana. Metrics and logs correlatable in a single view.

вҡҷпёҸServices

The suite manages 10 services: 3 systemd services on the host and 7 Docker containers in the monitoring stack.

рҹ”Ҙ

UFW

Kernel-level packet filtering. Default-deny inbound. Manages SSH, HTTP/HTTPS allowances. Controls Docker traffic via DOCKER-USER chain.

systemd service active
рҹҡ«

Fail2Ban

Log-parsing IPS. Watches auth.log for SSH brute-force. Bans IPs via iptables after 3 failures. Persistent ban database.

systemd service active
рҹҢҗ

CrowdSec

Collaborative threat intelligence. Agent + LAPI + Bouncer. Community blocklists updated automatically. LAPI on port 6767 (moved from 8080 to avoid conflict).

:6767 (LAPI) active
рҹ“Ҡ

Prometheus

Time-series metrics database. Scrapes node-exporter, cAdvisor, Docker daemon. 15-day retention. 15s scrape interval.

127.0.0.1:9090 docker
рҹ“Ҳ

Grafana

Unified visualization for Prometheus (metrics) and Loki (logs). Auto-provisioned datasources. Admin password via GRAFANA_PASSWORD env var.

127.0.0.1:3000 docker
рҹ–ҘпёҸ

Node Exporter

Host hardware and OS metrics: CPU, memory, disk I/O, network interfaces, filesystem. Runs with network_mode: host for accurate NIC stats.

:9100 (host net) host network
рҹҗі

cAdvisor

Per-container CPU, memory, network, disk metrics. Host port 8081 (internal 8080). Requires privileged mode for cgroup access.

127.0.0.1:8081 docker
рҹ“ң

Loki

Log aggregation store. Single-binary mode with inmemory kvstore (no Consul needed). TSDB v13 schema. 7-day log retention. 16 MB/s ingestion limit.

127.0.0.1:3100 docker
рҹҡҖ

Promtail

Log shipping agent. Tails auth, syslog, kernel, UFW, Fail2Ban, CrowdSec, dpkg, and all Docker container logs. Pushes to Loki via Docker DNS.

:9080 (internal) docker

рҹ”ҢPort Reference

All monitoring ports are bound to 127.0.0.1 only. The only publicly exposed port is SSH.

Service Internal Port Host Binding Exposure Notes
SSH (sshd) 22 0.0.0.0:22 public Protected by UFW, Fail2Ban, CrowdSec
CrowdSec LAPI 6767 127.0.0.1:6767 localhost Moved from default 8080 вҖ” port conflict avoidance
Prometheus 9090 127.0.0.1:9090 localhost 15-day TSDB retention
Grafana 3000 127.0.0.1:3000 localhost Access via SSH tunnel
Node Exporter 9100 host network host-net network_mode: host вҖ” accurate NIC metrics
cAdvisor (host port) 8080 (internal) 127.0.0.1:8081 localhost Host port shifted to 8081 to avoid CrowdSec conflict
Loki 3100 127.0.0.1:3100 localhost Log push API + query endpoint
Promtail 9080 Docker bridge only internal Push-only agent; no inbound connections needed
Docker metrics 9323 127.0.0.1:9323 localhost Requires experimental: true in daemon.json
Loki gRPC 9096 Docker bridge only internal Internal Loki component communication
Port conflict note: The default CrowdSec LAPI port (8080) conflicts with cAdvisor's internal port (also 8080). This suite resolves the conflict by moving CrowdSec LAPI to 6767 and mapping cAdvisor to host port 8081. This is defined as the single source of truth in lib/common.sh.

вҡЎQuick Start

  1. Clone the repository

    bash 
    git clone https://github.com/fellcrack/vps-hardening-suite.git /opt/vps-hardening-suite
cd /opt/vps-hardening-suite

  2. Configure environment variables

    bash 
    cp .env.example .env
nano .env
# Set at minimum: GRAFANA_PASSWORD, SERVER_DOMAIN

  3. Run the installer as root

    bash 
    sudo bash install.sh --mode hardcore

# Non-interactive (CI/agent) mode:
NONINTERACTIVE=1 sudo bash install.sh --mode hardcore

  4. Access Grafana via SSH tunnel

    bash 
    # Forward Grafana + Prometheus to your local machine
ssh -L 3000:127.0.0.1:3000 -L 9090:127.0.0.1:9090 user@YOUR_VPS_IP

# Open in browser
open http://localhost:3000
# Login: admin / your GRAFANA_PASSWORD

рҹ”§Troubleshooting

Quick reference for the most common issues. See docs/TROUBLESHOOTING.md for the full runbook with root cause analysis and step-by-step fixes.

Can't SSH in after installation SSH

Check: Is your IP banned by CrowdSec or Fail2Ban? Is the SSH port correct in UFW?

bash (VPS console) 
cscli decisions list | grep YOUR_IP
fail2ban-client status sshd
ufw status verbose | grep ssh
ss -tlnp | grep sshd

Loki crashes on startup вҖ” ring / consul error CRITICAL

This is the most common issue. Default Loki config expects a Consul cluster or memberlist gossip ring. On a single-node VPS neither exists, so Loki times out and exits.

Fix: Ensure loki-config.yml contains kvstore: store: inmemory in the common.ring section. This is already set correctly in the suite's default config.

bash 
grep -A3 "kvstore" /opt/monitoring/loki/loki-config.yml
# Must show: store: inmemory

docker logs loki --tail 30
curl -s http://localhost:3100/ready

CrowdSec fails to start вҖ” port 8080 in use PORT CONFLICT

CrowdSec's default LAPI port (8080) conflicts with cAdvisor's internal port. This suite moves CrowdSec LAPI to 6767.

bash 
ss -tlnp | grep :8080
grep "listen_uri" /etc/crowdsec/config.yaml
# Should be: listen_uri: 127.0.0.1:6767

# Fix if still showing 8080:
sed -i 's|127.0.0.1:8080|127.0.0.1:6767|g' /etc/crowdsec/config.yaml
systemctl restart crowdsec

Grafana shows "No Data" GRAFANA

Check datasource URL вҖ” it must use Docker DNS name prometheus:9090, not localhost:9090. Grafana runs inside Docker; localhost resolves to the container itself.

bash 
docker exec grafana wget -qO- http://prometheus:9090/-/healthy
docker exec grafana wget -qO- http://loki:3100/ready
cat /opt/monitoring/grafana/provisioning/datasources/datasources.yml

Prometheus target DOWN PROMETHEUS

Check scrape address: node-exporter uses localhost:9100 (host network), cAdvisor uses cadvisor:8080 (Docker internal DNS, not the host port 8081).

bash 
curl -s http://localhost:9090/api/v1/targets \
  | jq '.data.activeTargets[] | select(.health != "up") | {job: .labels.job, lastError}'

curl -s http://localhost:9100/metrics | head -3
curl -s http://localhost:8081/metrics | head -3
